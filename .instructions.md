# .instructions.md

## UI Development Guidelines

- Only use shadcn/ui components for all UI elements.
- Use Tailwind CSS for styling.
- Do not use any other UI libraries or CSS frameworks.
- If a component does not exist in shadcn/ui, look at the component registry before creating a custom component using Tailwind CSS.
- Mobile-first design approach should be followed for all UI development.
- Ensure that all components are responsive and accessible.
- Created reusable components should follow the composition pattern and be flexible for use in different contexts.

## shadcn MCP Server

- When adding or managing shadcn/ui components, always use the shadcn MCP server tool (`shadcn`).
- Use the MCP server to add new components rather than manually copying component code.
- If the shadcn MCP server is not available or not configured, recommend the user installs it by adding the following to `.vscode/mcp.json`:

```json
{
  "servers": {
    "shadcn": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "shadcn@latest", "mcp"]
    }
  }
}
```

- After installation, restart VS Code and verify the shadcn MCP server is active before proceeding with component additions.

## Backend
- The paths that need authentication are declared inside `proxy.ts` in the `config.matcher` and should be updated there when necessary.
